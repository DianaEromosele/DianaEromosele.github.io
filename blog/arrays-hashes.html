Arrays Are For Single Objects, Hashes Are Pairs That Are Married

They're both containers filled with intel, but the intel is stored and called upon in different ways.

Think of an "array" as a classroom that contains individuals who are single, and hashes as classrooms that contain pairs who are married--inextricably linked.

The array classroom can hold a variety of objects such as string, integers, other arrays and even hashes. Each of those objects (or single person), is in a specific spot. That's how you call on that single person. Diana is in slot 4, Mary is in slot 2, Eric is in slot 10, and so on and so forth. Single people can leave the array, or come back into the array, but even as all of this transition happens, objects are moved from slot to slot.

This is how an array looks like:

``` array = ["Diana", 25, "Elizabeth", 2, "Kate", "Prince Charles"] ```

To retrieve a specific piece of data in an array, it's helpful to know what slot that data is in. In our array above, Elizabeth is in slot 2, (arrays start with the number "0" in the first slot all the way to the left). Calling Elizabeth would look like this: array[2]. In a hash, the classroom has married folk. One person is the key (let's say the husband is), and the other is the value, the wife. You call on the husband, you get the wife. What hashes come together, let no man put asunder (lol). Here is what a hash looks like

```` couples = { "jay-z": "beyonce", "barack":"michelle", "prince_charles": "kate middleton", "will": "jada", "brad": angelina } ````

If I wanted to call on Beyonce, I would use this syntax: couples["jay-z"]. As you can see, hashes are bit more intuitive, meaning, if you have knowledge about the key, and you know what value is attached to it, then it's as easy as calling on that key. Similar to storing someone's address in a system. To look up Diana's address, call on the Diana key in the "Address" Hash, and that value will pop up.
