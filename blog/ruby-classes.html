Classism Isn't a Bad Thing in Programming

It gives stuff order, power and restores order.

Let's create a "Diana" Class.

This class will hold instance variables, about myself, like my name, age and my career. I'll then create a couple of methods that will allow me to use those instance variables. The reason why I created instance variables is that I can use the in a variety of methods inside this class. Take a look below.

class Diana

def initialize(name, age, career)

@name = name

@age = 29

@career = software engineer

end

def how_she_greets

puts "Hey my name is #{@name}. i'm #{@age}. nice to meet you."

end

def how_she_networks

puts "i'm a #{career} in training. My background is in media; I used to be a journalist."

end end
